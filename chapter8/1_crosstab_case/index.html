8.1 CASE文によるクロス集計

GROUP BYとCASE文を組み合わせた王道テクニック

「クロス集計」とは、縦軸と横軸にそれぞれ項目を取り、その交点に集計値を表示する表のことです。SQLでこれを実現する最も古典的で強力な方法が、GROUP BYとCASE文の組み合わせです。

基本的な考え方

  1. 縦軸にしたい項目（例: 商品カテゴリ）でGROUP BYする。
  2. 横軸にしたい項目（例: 販売月）の各値について、CASE文で条件分岐させる。
  3. CASE文全体を集計関数（SUMやCOUNT）で囲み、各交点の値を計算する。

使用例：カテゴリ別・月別の売上クロス集計表

SalesテーブルとProductsテーブルがあると仮定し、「商品カテゴリごとの月別売上合計」を集計してみましょう。

SELECT
  p.Category,
  -- 4月の売上合計
  SUM(CASE WHEN strftime('%m', s.SaleDate) = '04' THEN s.Amount ELSE 0 END) AS "4月",
  -- 5月の売上合計
  SUM(CASE WHEN strftime('%m', s.SaleDate) = '05' THEN s.Amount ELSE 0 END) AS "5月",
  -- 6月の売上合計
  SUM(CASE WHEN strftime('%m', s.SaleDate) = '06' THEN s.Amount ELSE 0 END) AS "6月",
  -- カテゴリごとの総合計
  SUM(s.Amount) AS "総合計"
FROM
  Sales AS s
JOIN
  Products AS p ON s.ProductID = p.ProductID
WHERE
  strftime('%Y', s.SaleDate) = '2024'
GROUP BY
  p.Category
ORDER BY
  p.Category;

ポイント:

  • SUM(CASE ... END)の部分がキモです。「もし4月のデータなら金額を、そうでなければ0を返し、それをカテゴリごとに合計する」という処理を行っています。
  • これにより、行（GROUP BYの対象）と列（CASE文の条件）が交差するセルの値を計算できます。

この手法は、横軸の項目が増えてもSUM(CASE ...)の部分をコピーして修正するだけで対応できるため、非常に応用範囲が広いです。
