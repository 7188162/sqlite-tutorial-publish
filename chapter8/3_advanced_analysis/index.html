8.3 CTEを活用した段階的集計

複雑な分析は、CTEでステップに分解して考える

クロス集計のさらに一歩先へ進み、「カテゴリ別の月次売上と、その売上が全社合計に占める構成比」を同時に算出する、といった複雑な分析に挑戦します。このような多段階の集計は、CTE（WITH句）の真骨頂です。

分析のステップ

  1. ステップ1: まず、月別・カテゴリ別の売上合計を計算する。（中間サマリ）
  2. ステップ2: 次に、月別の全社売上合計を計算する。
  3. ステップ3: 最後に、ステップ1とステップ2の結果を結合し、構成比（カテゴリ別売上 ÷ 全社合計売上）を計算する。

この思考プロセスを、そのままCTEを使ったSQLに落とし込みます。

-- ステップ1: 月別・カテゴリ別の売上集計
WITH MonthlyCategorySales AS (
  SELECT
    strftime('%Y-%m', SaleDate) AS SaleMonth,
    p.Category,
    SUM(s.Amount) AS CategoryAmount
  FROM Sales AS s
  JOIN Products AS p ON s.ProductID = p.ProductID
  GROUP BY SaleMonth, p.Category
),
-- ステップ2: 月別の全社売上合計
MonthlyTotalSales AS (
  SELECT
    SaleMonth,
    SUM(CategoryAmount) AS TotalAmount
  FROM MonthlyCategorySales
  GROUP BY SaleMonth
)
-- ステップ3: 2つのCTEを結合し、最終レポートを作成
SELECT
  mcs.SaleMonth,
  mcs.Category,
  mcs.CategoryAmount,
  mts.TotalAmount AS MonthlyTotalAmount,
  -- 構成比を計算 (小数点以下を扱うため100.0を掛ける)
  (mcs.CategoryAmount * 100.0 / mts.TotalAmount) AS ShareRatio
FROM
  MonthlyCategorySales AS mcs
JOIN
  MonthlyTotalSales AS mts ON mcs.SaleMonth = mts.SaleMonth
ORDER BY
  mcs.SaleMonth, mcs.CategoryAmount DESC;

このように、CTEを使って処理を論理的なステップに分解することで、一見複雑に見える分析も、一つ一つの簡単なクエリの組み合わせとして構築できます。このアプローチは、後からSQLを修正したり、他の人がコードを読んだりする際のメンテナンス性を劇的に向上させます。
