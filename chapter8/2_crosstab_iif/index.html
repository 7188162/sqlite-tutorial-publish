8.2 IIF関数によるシンプルなクロス集計

CASE文よりも簡潔に記述する方法

前セクションのCASE文は強力ですが、条件分岐が単純な場合はIIF関数を使うと、よりスッキリと記述できます。

IIF関数を使ったクロス集計

CASE WHEN 条件 THEN 値 ELSE 0 END の部分を、IIF(条件, 値, 0) に置き換えるだけです。

SELECT
  p.Category,
  SUM(IIF(strftime('%m', s.SaleDate) = '04', s.Amount, 0)) AS "4月",
  SUM(IIF(strftime('%m', s.SaleDate) = '05', s.Amount, 0)) AS "5月",
  SUM(IIF(strftime('%m', s.SaleDate) = '06', s.Amount, 0)) AS "6月",
  SUM(s.Amount) AS "総合計"
FROM
  Sales AS s
JOIN
  Products AS p ON s.ProductID = p.ProductID
WHERE
  strftime('%Y', s.SaleDate) = '2024'
GROUP BY
  p.Category
ORDER BY
  p.Category;

CASE文を使った例と結果は全く同じですが、SQLの記述が少し短くなりました。 このような単純な条件分岐の集計では、IIF関数を積極的に活用すると良いでしょう。

COALESCEによるNULL値の処理

SUMやCOUNTなどの集計関数は、対象となる行がない場合にNULLを返すことがあります。表計算ソフトのように0を表示したい場合は、COALESCE関数で囲むのがスマートです。
