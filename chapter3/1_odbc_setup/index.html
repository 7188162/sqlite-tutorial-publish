3.1 ODBCドライバのセットアップ

AccessとSQLiteを繋ぐための「通訳」を準備します

AccessとSQLiteは、本来は直接会話することができません。そこで、両者の間に入って言葉を翻訳してくれる「ODBC (Open Database Connectivity)ドライバ」という通訳をPCに導入します。

SQLite ODBCドライバのダウンロード

  1. SQLite ODBC Driverの公式サイト🔗 にアクセスします。
  2. 「Download」セクションから、お使いのWindowsに合ったインストーラーをダウンロードします。通常は「sqliteodbc_w64.exe」（64ビット版）で問題ありません。

[画像: SQLite ODBC Driverのダウンロードページ。該当ファイルを赤枠で囲む]

インストールと設定

  1. ダウンロードしたインストーラーを実行し、画面の指示に従ってインストールします。特に設定の変更は不要です。
  2. インストールが完了したら、WindowsのODBC設定を確認します。
    • Windowsの検索バーで「ODBC」と入力し、「ODBC データ ソース (64ビット)」を起動します。

[画像: Windowsの検索バーでODBCを検索する様子]

  1. 「ODBCデータソースアドミニストレーター」が開きます。「ドライバー」タブをクリックしてください。
  2. 一覧の中に「SQLite3 ODBC Driver」という名前があれば、正常にインストールされています。

[画像: ODBCデータソースアドミニストレーターの「ドライバー」タブ。SQLite3 ODBC Driverをハイライト]

DSNの登録は不要

ODBCには「データソース名(DSN)」を事前に登録する方法もありますが、Accessから直接接続する場合は不要です。ここではドライバがPCにインストールされていることだけ確認できればOKです。

これで、AccessがSQLiteと会話するための準備が整いました。
