3.3 パススルークエリで性能を最大化

重い処理はSQLiteに直接命令して高速化しよう

リンクテーブルは非常に便利ですが、Access上で複雑なクエリ（特に集計や複数テーブルのJOIN）を実行すると、Accessのクエリエンジンがデータを一つ一つSQLiteに問い合わせるため、非効率になることがあります。

そこで登場するのが「パススルー クエリ」です。これは、Accessのエンジンを介さず、SQL文をそのままSQLiteに直接送りつけて実行させる機能です。これにより、SQLite本来の高速な処理能力を最大限に引き出すことができます。

パススルークエリの作成方法

  1. Accessのリボンで「作成」タブ → 「クエリ デザイン」をクリックします。
  2. 「テーブルの表示」ウィンドウはすぐに閉じます。
  3. デザインビューの上部にある「パススルー」アイコンをクリックします。

[画像: Accessのクエリデザインビュー。「パススルー」アイコンをハイライト]

  1. SQLビューに切り替わります。ここに、SQLiteの構文でSQLを記述します。
  2. プロパティシート（F4キーで表示）を開き、「ODBC 接続文字列」を設定します。
    • 「…」ボタンをクリックし、前セクションで設定した時と同様に、データソース（SQLite3 ODBC Driverとデータベースファイル）を選択します。
    • 一度設定すると、ODBC;DRIVER=SQLite3 ODBC Driver;Database=C:\MyData\sales.db;... のような文字列が自動で入力されます。

[画像: パススルークエリのプロパティシート。ODBC接続文字列の設定箇所をハイライト]

パススルークエリの活用例

例えば、「月別の売上合計」を計算する重い処理を考えます。

-- このSQLはAccessではなく、SQLite側で実行される
WITH monthly_sales AS (
    SELECT
        strftime('%Y-%m', 販売日) AS 販売月,
        SUM(金額) AS 合計金額
    FROM
        T_売上
    GROUP BY
        strftime('%Y-%m', 販売日)
)
SELECT
    販売月,
    合計金額
FROM
    monthly_sales
ORDER BY
    販売月;
重要：構文の違いに注意！

パススルークエリ内では、Accessの関数（Format()など）は使えません。SQLiteの関数（strftime()など）を使う必要があります。これは、SQLが直接SQLiteに送られるためです。

このクエリを実行すると、計算はすべてSQLite側で一括して行われ、Accessはその結果だけを受け取ります。データ件数が数万件以上の場合、リンクテーブルに対してAccessのクエリを実行するよりも劇的に高速化されることを体感できるでしょう。
