7.3 効果の測定：EXPLAIN QUERY PLAN

インデックスが本当に使われているかを確認する最も重要な方法

インデックスを作成しても、SQLiteがそれをクエリ実行時に利用してくれなければ意味がありません。インデックスが使われているかを確認するには、EXPLAIN QUERY PLANという魔法のコマンドを使います。

使い方は簡単で、確認したいSELECT文の先頭にEXPLAIN QUERY PLANを付けるだけです。

使用例

インデックスがない状態での確認 CustomerID列にインデックスがない状態で、以下のクエリを実行します。

EXPLAIN QUERY PLANの実行
1
2
3


EXPLAIN QUERY PLAN
SELECT * FROM Sales WHERE CustomerID = 'C1025';

実行結果

detail
SCAN TABLE Sales

このSCAN TABLEが「フルテーブルスキャン」を意味します。つまり、テーブルを最初から最後まで全部探している状態で、非常に非効率です。

インデックスを作成し、再度確認 CustomerID列にインデックス IX_Sales_CustomerID を作成した上で、もう一度同じコマンドを実行します。

実行結果

detail
SEARCH TABLE Sales USING INDEX IX_Sales_CustomerID (CustomerID=?)

SEARCH TABLE ... USING INDEX ...と表示されれば成功です！これは、SQLiteがIX_Sales_CustomerIDというインデックスを使って効率的にデータを検索していることを示しています。

A5 SQL:Mk2での実行計画確認

A5 SQL:Mk2では、SQLエディタ上でクエリを選択し、F6キーを押すか、ツールバーの「実行計画の表示」ボタンをクリックするだけで、簡単に実行計画を確認できます。

WHERE句やORDER BY句を含むクエリのパフォーマンスチューニングを行う際は、必ずこのEXPLAIN QUERY PLANでインデックスが有効活用されているかを確認する癖をつけましょう。
