7.2 インデックスの作成方法

SQLとGUIの両方からインデックスを追加します

SQLによる作成

CREATE INDEX文を使います。インデックス名、対象テーブル名、対象列名を指定するのが基本です。

CREATE INDEX構文
1
2
3
4
5

-- SalesテーブルのCustomerID列にインデックスを作成
CREATE INDEX IX_Sales_CustomerID ON Sales (CustomerID);

-- 同じくSalesテーブルのSaleDate列にインデックスを作成
CREATE INDEX IX_Sales_SaleDate ON Sales (SaleDate);
インデックス名は、IX_テーブル名_列名 のように、分かりやすい命名規則を決めておくと管理がしやすくなります。

A5 SQL:Mk2 (GUI) による作成

  1. データベースエクスプローラで、対象のテーブル（例: Sales）を展開し、「インデックス」フォルダを右クリックして「インデックスの作成」を選択します。

[画像: A5 SQL:Mk2のテーブルツリー。「インデックスの作成」をハイライト]

  1. インデックス作成ウィンドウが開きます。
    • インデックス名: IX_Sales_ProductID のように入力します。
    • 構成カラム: テーブルの列一覧から、インデックスに含めたい列（例: ProductID）を選択し、「>」ボタンで右側の「インデックス構成カラム」に移動します。

[画像: インデックス作成ウィンドウ。設定項目をハイライト]

  1. 「実行(F5)」ボタンを押すとインデックスが作成されます。

作成済みインデックスの確認

sqlite_masterという特別なテーブルを検索することで、データベース内に存在するすべてのインデックスを確認できます。

SELECT
  name,       -- インデックス名
  tbl_name,   -- テーブル名
  sql         -- インデックス作成時のSQL
FROM
  sqlite_master
WHERE
  type = 'index' AND tbl_name NOT LIKE 'sqlite_%';
