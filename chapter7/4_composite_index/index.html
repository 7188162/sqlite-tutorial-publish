7.4 複合インデックスの活用

複数の列を組み合わせた検索を高速化します

WHERE句で複数の列を条件に指定することがよくあります。このような場合に効果を発揮するのが、複数の列をまとめて一つのインデックスにした「複合インデックス」です。

複合インデックスの作成

作成方法は簡単で、CREATE INDEX文の列指定で複数の列をカンマで区切るだけです。

複合インデックスの作成
1
2
3


-- Category列とBrand列で商品を検索するクエリを高速化したい
CREATE INDEX IX_Products_Category_Brand ON Products (Category, Brand);

複合インデックスが有効なケース

上記のインデックスは、以下の様なクエリで非常に効果的です。

複合インデックスが使われるクエリ
1
2
3
4
5
6


-- 列の順番が同じ場合 (最も効果的)
SELECT * FROM Products WHERE Category = '文房具' AND Brand = 'コクヨ';

-- インデックスの先頭列だけを使う場合 (効果あり)
SELECT * FROM Products WHERE Category = '文房具';

注意点：列の順番が重要

複合インデックスは、指定した列の順番が非常に重要です。インデックスは左側の列から順にしか使われません。

上記のON Products (Category, Brand)というインデックスは、以下の様なクエリでは効果がありません。

複合インデックスが使われないクエリ
1
2
3


-- インデックスの2番目の列だけでは使えない
SELECT * FROM Products WHERE Brand = 'コクヨ';
Categoryを指定せずにBrandだけを指定しても、索引の先頭がCategoryでソートされているため、うまく活用できないのです。

設計のポイント

複合インデックスを設計する際は、WHERE句で指定される頻度が高い列、そしてカーディナリティ（値のばらつき）が高い列を、インデックスの左側に配置するのがセオリーです。
