6.1 CTE (WITH句) による可読性の向上

SQLをパーツに分解して、処理の流れを明確にします

CTE（Common Table Expression: 共通テーブル式）は、WITH句を使って、クエリの中で一時的に利用できる名前付きの結果セットを定義する機能です。複雑なクエリを、意味のある部品（パーツ）に分解することで、驚くほど可読性が向上します。

WITH <CTE名1> AS (
  -- ここにSELECT文を記述
),
<CTE名2> AS (
  -- 先に定義したCTE名1を利用することも可能
)
-- メインのクエリ
SELECT ... FROM <CTE名1> ...

使用例：サブクエリの書き換え

例えば、「部門ごとの売上合計を算出し、全社平均売上を超えている部門だけをリストアップする」という処理を考えます。

SELECT
  DepartmentName,
  TotalSales
FROM (
  -- まず部門ごとの売上合計を計算
  SELECT
    d.DepartmentName,
    SUM(s.Amount) AS TotalSales
  FROM Sales AS s
  JOIN Employees AS e ON s.EmployeeID = e.EmployeeID
  JOIN Departments AS d ON e.DepartmentID = d.DepartmentID
  GROUP BY d.DepartmentName
) AS DepartmentSales
WHERE
  TotalSales > (SELECT AVG(TotalSales) FROM (
    -- 平均を出すためにもう一度同じ計算...
    SELECT SUM(s.Amount) AS TotalSales
    FROM Sales AS s
    JOIN Employees AS e ON s.EmployeeID = e.EmployeeID
    GROUP BY e.DepartmentID
  ));

WITH DepartmentSales AS (
  -- 部品1: 部門ごとの売上合計を計算
  SELECT
    d.DepartmentName,
    SUM(s.Amount) AS TotalSales
  FROM Sales AS s
  JOIN Employees AS e ON s.EmployeeID = e.EmployeeID
  JOIN Departments AS d ON e.DepartmentID = d.DepartmentID
  GROUP BY d.DepartmentName
)
-- メイン処理: 部品1を使って、平均超の部門を抽出
SELECT
  DepartmentName,
  TotalSales
FROM
  DepartmentSales
WHERE
  TotalSales > (SELECT AVG(TotalSales) FROM DepartmentSales);

CTEを使うと、処理のステップが上から下に流れるようになり、「まず何をして、次に何をするか」が一目瞭然になります。また、同じ計算処理を何度も書く必要がなくなり、再利用性が高まるのも大きなメリットです。
