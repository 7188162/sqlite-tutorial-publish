6.3 IF EXISTSによる安全な操作

オブジェクトの存在を確認してから処理を実行します

テーブルやビューを削除（DROP）したり、作成（CREATE）したりする際に、「すでに存在する場合」や「存在しない場合」にエラーが発生するのを防ぐための構文です。

DROP ... IF EXISTS

オブジェクトが存在する場合にのみ、削除を実行します。バッチ処理などで、毎回まっさらな状態からテーブルを作成したい場合などによく使われます。

テーブルが存在すれば削除する

-- もし T_MonthlySummary というテーブルが存在すれば、それを削除する
DROP TABLE IF EXISTS T_MonthlySummary;

-- この後で、新しい T_MonthlySummary テーブルを作成する処理を続ける
CREATE TABLE T_MonthlySummary AS ...;
これがあれば、初回実行時（テーブルが存在しない）でもエラーにならず、安全にスクリプトを実行できます。

CREATE ... IF NOT EXISTS

オブジェクトが存在しない場合にのみ、作成を実行します。

ビューが存在しなければ作成する
1

-- もし V_SalesDetails というビューが存在しなければ、それを作成する
CREATE VIEW IF NOT EXISTS V_SalesDetails AS
SELECT ...
FROM ...;
A5 SQL:Mk2の挙動

A5 SQL:Mk2のテーブル作成GUIなどで生成されるSQLには、デフォルトで IF NOT EXISTS が付いていることが多いです。ツールが安全なコードを自動生成してくれている良い例です。

これらの構文を適切に使うことで、誰がいつ実行してもエラーが起きにくい、堅牢なSQLスクリプトを作成することができます。
