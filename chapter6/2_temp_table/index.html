6.2 一時テーブルの活用

複雑な集計の中間結果を保存して、処理を分割します

CTEが単一のクエリ内でのみ有効なのに対し、「一時テーブル」はデータベース接続が続いている間、ずっと利用できる一時的なテーブルです。非常に大規模で複雑な集計を、複数のステップに分けて実行する場合に役立ちます。

一時テーブルの作成

CREATE TEMP TABLE または CREATE TEMPORARY TABLE を使って作成します。

CREATE TEMP TABLE T_MonthlySummary AS
SELECT
  strftime('%Y-%m', SaleDate) AS SaleMonth,
  ProductID,
  SUM(Quantity) AS TotalQuantity,
  SUM(Quantity * UnitPrice) AS TotalAmount
FROM
  Sales
GROUP BY
  SaleMonth, ProductID;
これで、T_MonthlySummary という名前の一時テーブルが作成され、月別・製品別の集計結果が格納されます。このテーブルは、A5 SQL:Mk2のデータベースエクスプローラ上にも（通常とは別のアイコンで）表示されます。

一時テーブルの利用

一度作成すれば、通常のテーブルと全く同じように SELECT や JOIN ができます。

-- 作成した月次サマリ（一時テーブル）から、月ごとのトップ3製品を抽出する
WITH RankedSales AS (
  SELECT
    SaleMonth,
    ProductID,
    TotalAmount,
    RANK() OVER (PARTITION BY SaleMonth ORDER BY TotalAmount DESC) AS Rank
  FROM
    T_MonthlySummary
)
SELECT
  SaleMonth,
  ProductID,
  TotalAmount
FROM
  RankedSales
WHERE
  Rank <= 3;
CTEと一時テーブルの使い分け

  • CTE: 1つのクエリ内で完結する、参照のみのロジックの分割に使います。SQLが読みやすくなるのが主目的です。
  • 一時テーブル: 複数のクエリにまたがって中間結果を再利用したい場合や、中間結果にインデックスを作成してパフォーマンスを改善したい場合に使います。
