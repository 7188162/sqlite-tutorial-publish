4.1 テーブルの作成

A5 SQL:Mk2のGUIで直感的にテーブルを設計します

データベースにデータを格納するには、まず器となる「テーブル」を作成する必要があります。

  1. A5 SQL:Mk2のデータベースエクスプローラで、対象のデータベース（例: 売上管理DB）を右クリックし、「テーブルの作成」を選択します。

[画像: A5 SQL:Mk2の右クリックメニュー。「テーブルの作成」をハイライト]

  1. テーブル作成ウィンドウが開きます。Accessのテーブルデザインビューによく似た画面です。
  2. 画面下部のグリッドに、テーブルに含めたい列（フィールド）の情報を入力していきます。
    • 物理名: 列の物理的な名前です。通常は半角英数字を使います（例: SaleID, SaleDate, ProductID）。
    • 論理名: 列の日本語名や説明です（例: 売上ID, 販売日, 商品ID）。A5 SQL:Mk2が自動でコメントとしてSQLに追加してくれます。
    • データ型: INTEGER (整数), REAL (数値), TEXT (文字列), BLOB (バイナリ) から選択します。日付や時刻も TEXT や INTEGER で格納するのが一般的です。
    • Not Null: Accessの「値要求」に相当します。チェックすると、この列に空の値（NULL）を入れることができなくなります。
    • PK: 主キー（Primary Key）に設定したい列にチェックを入れます。

[画像: テーブル作成ウィンドウ。各項目を入力している様子]

  1. ウィンドウ上部の「テーブル物理名」に、テーブルの名前（例: Sales）を入力します。
  2. 設定が完了したら、右下の「実行(F5)」ボタンを押すと、テーブルが作成されます。
SQLプレビュー

画面右側の「DDL」タブをクリックすると、GUIでの設定内容がどのような CREATE TABLE 文として実行されるかを確認できます。GUI操作とSQLの対応を学ぶのに非常に役立ちます。
