4.3 ビューの作成（Accessクエリの代替）

よく使うSELECT文を「ビュー」として保存し、再利用します

MS Accessで多用する「選択クエリ」は、SQLiteでは「ビュー(View)」という機能で代替します。ビューとは、SELECT文に名前を付けてデータベース内に保存した、仮想的なテーブルのことです。

ビューの作成方法

  1. データベースエクスプローラで「ビュー」フォルダを右クリックし、「ビューの作成」を選択します。

[画像: データベースエクスプローラの「ビュー」を右クリック]

  1. ビュー作成ウィンドウが開きます。
    • ビュー名: 作成するビューの名前を入力します（例: V_MonthlySales）。
    • SQL入力エリア: ここに、保存したいSELECT文を記述します。

[画像: ビュー作成ウィンドウ。ビュー名とSQL入力エリアを指し示す]

ビューの作成例

例えば、商品テーブル（Products）と売上テーブル（Sales）を結合し、商品名を含んだ売上明細を表示する、というよく使うクエリをビューにしてみましょう。

CREATE VIEW V_SalesDetails AS
SELECT
  s.SaleID,
  s.SaleDate,
  p.ProductName,
  s.Quantity,
  s.UnitPrice,
  s.Quantity * s.UnitPrice AS Subtotal
FROM
  Sales AS s
LEFT JOIN
  Products AS p ON s.ProductID = p.ProductID;
A5 SQL:Mk2のクエリビルダ

SQLに慣れていないうちは、A5 SQL:Mk2の「クエリビルダ」機能を使うと、AccessのクエリデザインビューのようにGUIでテーブルを結合し、SQLを自動生成できます。

上記のSQLを貼り付けて「実行(F5)」ボタンを押すと、ビューが作成されます。

ビューの利用

作成されたビューは、データベースエクスプローラ上ではテーブルとほぼ同じように扱えます。

  • ビューをダブルクリックすれば、結果をいつでも確認できます。
  • 他のクエリから、まるで本物のテーブルのようにこのビューを FROM V_SalesDetails のように参照できます。

Accessのクエリを多用していた方にとって、このビューは最も重要な機能の一つです。積極的に活用していきましょう。
