5.1 SELECT文とWHERE句の基本

データ抽出の第一歩と、効率的な絞り込み

SELECT文とWHERE句は、SQLの最も基本となる要素です。ここでは簡単におさらいし、パフォーマンスに関するヒントに触れます。

SELECT文：列の指定

SELECT句には、取得したい列名をカンマ区切りで指定します。*（アスタリスク）は全列を意味しますが、必要な列だけを指定する方が効率的です。

必要な列だけを指定する例
1
2
3
4
5
6


-- 非推奨：全列を取得
SELECT * FROM Sales;

-- 推奨：必要な列だけを指定
SELECT SaleDate, ProductID, Quantity FROM Sales;

WHERE句：行の絞り込み

WHERE句は、取得する行の条件を指定します。条件を指定する列に「インデックス」が設定されていると、検索が劇的に高速になります（インデックスについては第7章で詳しく解説します）。

特定の期間のデータを抽出
1
2
3
4
5


-- 2024年1月1日から1月31日までの売上データを抽出
SELECT SaleDate, Quantity, UnitPrice
FROM Sales
WHERE SaleDate >= '2024-01-01' AND SaleDate <= '2024-01-31';
関数を使った条件指定の注意点

WHERE strftime('%Y', SaleDate) = '2024' のように、WHERE句の列側に関数を適用すると、インデックスが利用されず、パフォーマンスが低下する原因になります。可能な限り、列はそのままの形で比較するのがセオリーです。
