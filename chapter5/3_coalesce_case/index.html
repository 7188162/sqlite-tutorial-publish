5.3 COALESCEとCASE文

NULL値の処理と、より複雑な条件分岐

IIF関数は便利ですが、NULLの扱いや3つ以上の複雑な条件分岐には対応しきれません。ここでは、それらを解決するCOALESCE関数とCASE文を学びます。

COALESCE関数：NULL値を別の値に置換

COALESCE(値1, 値2, 値3, ...)は、引数を左から順番に評価し、最初に見つかったNULLでない値を返します。これは、NULLかもしれない列にデフォルト値を与えたい場合に非常に便利です。

例えば、顧客テーブルの「備考」列がNULLの場合に「（備考なし）」と表示したいケースを考えます。

SELECT
  CustomerID,
  CustomerName,
  COALESCE(Remarks, '（備考なし）') AS DisplayRemarks
FROM
  Customers;
もしRemarks列に何か値が入っていればその値が、NULLであれば '（備考なし）' が返されます。

CASE文：複雑な条件分岐

IIF関数が2択の分岐しかできないのに対し、CASE文は何段階もの条件分岐を記述できます。

構文:

CASE
  WHEN 条件1 THEN 結果1
  WHEN 条件2 THEN 結果2
  ...
  ELSE 結果N
END

使用例：売上金額に応じた詳細なランク付け

SELECT
  SaleID,
  Quantity * UnitPrice AS Amount,
  CASE
    WHEN Quantity * UnitPrice >= 50000 THEN 'プラチナ'
    WHEN Quantity * UnitPrice >= 10000 THEN 'ゴールド'
    WHEN Quantity * UnitPrice >= 3000 THEN 'シルバー'
    ELSE 'ブロンズ'
  END AS Rank
FROM
  Sales;
IIFとCASEの使い分け

  • IIF: 条件が単純な「AまたはB」の2択の場合。記述がシンプルです。
  • CASE: 条件が3つ以上ある場合、または条件式が複雑な場合。可読性が高く、拡張性もあります。 迷ったら、より汎用的なCASE文を使うと覚えておくと良いでしょう。
